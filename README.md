# 🌟 Hoshi

Hoshi is an offline-first digital companion that grows alongside its user. It is not a chatbot or a productivity tool, but a living simulation of a small digital creature that evolves based on how you use your computer.

Hoshi runs locally, is lightweight, privacy-first, and designed to work on modest hardware without requiring constant internet access or cloud dependencies.

---

## ✨ Core Idea

Hoshi simulates a living companion that:

- Develops personality over time
- Forms memories based on user activity
- Reacts to applications, behavior, and system events
- Evolves emotionally and behaviorally
- Exists as a persistent presence on your desktop

AI is optional and only enhances specific interactions. The core system is fully deterministic and runs locally.

---

## 🧠 Key Principles

- Offline-first
- Low-resource execution
- Privacy by design
- No mandatory cloud dependencies
- Modular architecture
- Extensible companion behavior system

---

## 🎮 What Hoshi does

Hoshi is always “alive” in the background:

- Observes app usage patterns (locally)
- Reacts with animations and behaviors
- Builds memory fragments of user activity
- Develops preferences over time
- Changes emotional state dynamically
- Interacts occasionally through dialogue
- Evolves visually and behaviorally

---

## 🧩 Architecture Overview

Hoshi is built as a modular engine composed of:

- Behavior Engine (state + decision system)
- Emotion Engine (dynamic emotional state model)
- Memory Engine (event storage + retrieval)
- Identity Engine (personality & traits evolution)
- Animation Engine (2D sprite/skeleton system)
- System Observer (OS-level activity tracking)
- AI Layer (optional LLM integration via BYOK)
- Plugin System (future extensibility)

---

## ⚙️ Tech Stack (planned)

- Electron / Tauri (desktop runtime)
- React (UI layer)
- TypeScript (core logic)
- SQLite (local memory storage)
